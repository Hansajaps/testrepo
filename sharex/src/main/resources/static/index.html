🔄 1. Leader Election

Nodes use randomized election timeouts (150-300ms) to trigger elections. A Candidate becomes Leader by receiving majority votes in a specific 'Term'.

if (electionTimeout) {
  becomeCandidate();
  requestVotes();
}

📝 2. Log Replication

The Leader accepts client commands, appends them to its log, and replicates them to Followers via AppendEntries RPCs. Committed once majority acknowledges.

replicate(entry);
if (majorityAck) {
  commitEntry(index);
}

⚖️ Safety Properties

  • Election Safety: At most one leader per term.
  • Log Matching: Continuous log continuity check.
  • Leader Completeness: Leader has all committed entries.

🛡️ Fault Tolerance

  • Automatic Failover: Rapid re-election on leader loss.
  • ZK Fallback: Operates even if ZooKeeper fails.
  • State Recovery: Persistent log for crash recovery.
